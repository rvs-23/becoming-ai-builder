## Key explanations

- **Clone vs fork.** Clone makes a local copy that points back to my repo; fork first makes a copy under *your* GitHub account, so you'd maintain two repos. We clone (no fork) for this; forking is for contributing to open-source projects you don't own.
- **Arcs.** The year is split into arcs (mini-projects). Each arc is one or more sessions. The first session of an arc is usually an intro like today.
- **The new way of working.** Delivery and review stay; the "writing code by hand" middle shrinks and the "thinking and reviewing" part grows. Less typing, more thinking.
- **Three AI modes** (I'll always say which): AI off (basics, showing what you know), AI hints (when stuck, after you say what you think is wrong), AI loud (building real projects). Allowed models: **Gemini Flash** or **Claude Haiku**, web app only, no API key. No Codex, Claude Code, or Cline.
- **The core rule:** if you can't explain it, it isn't yours. You can't ask AI to defend AI's work, you're the one accountable.
- **CSV.** A file format for tables: first line is the header (column names), each line after is a row, values separated by commas.
- **CLI.** To ask a computer a question and get an answer, your options are a website, a windowed app, or the terminal. The terminal is the smallest to build, so our project is a command-line interface (like git).
- **The project goal:** a CLI that answers questions about NBA player stats, e.g. "How many points did KD score?", "Who's better, X or Y?", "Who leads the league in rebounds?"

## Improvements for Prateek

From the basketball-teaching exercise (this is coaching for explaining *and* prompting):

- **Start with the goal.** Open with what the thing is and how you win, then the details.
- **Don't assume.** Spell out the basics a beginner wouldn't know: how the winner is decided, the time limit, players per side, whether one position maps to one player.
- **Avoid "obviously."** It isn't obvious to a beginner, or to an AI you're prompting.
- **Teaching = understanding.** Your second explanation was far clearer than the first; that gap is the skill we're building.
