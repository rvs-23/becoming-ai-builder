## Key explanations

- **Version control**
  - Clone copies the repo locally but still points back to my GitHub repo; fork makes a separate copy under your account (two repos to maintain). We clone; fork is for open-source projects you don't own.
- **CSV**
  - A table as plain text: the first line is the header (column names), each line after is a row, values separated by commas.
- **CLI**
  - To ask a computer a question you could build a website, a windowed app, or use the terminal. The terminal needs the least setup, so our project is a command-line tool like git.
- **Working with AI**
  - Less code typed by hand, more thinking and reviewing. The "write the code" step shrinks; the "review it" step grows.
- **Explaining as a skill**
  - Teaching something forces you to understand it, and it is the same skill as prompting an AI clearly.
- **Basketball as data**
  - The stats in our CSV are game events made countable: rebounds, assists, steals, blocks, turnovers, points.
