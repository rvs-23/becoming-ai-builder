## Key explanations

- **Fork vs clone**
  - Cloning makes a local copy that still points back to my GitHub repo. Forking first makes its own copy under your account, so you would manage two repos. We clone; forking is for contributing to open-source projects you don't own.
- **CSV**
  - A way to store a table as plain text. The first line is the header (the column names), every line after it is one row, and commas separate the values.
- **CLI**
  - We wanted to ask the computer a question and get an answer back. We listed the options and eliminated each: a website and a windowed app both need a lot of building, so we ruled them out. The terminal needs the least setup, so our project is a command-line tool, like git.
- **Working with AI**
  - You type less code by hand and spend more time thinking and reviewing what the AI wrote. The writing step shrinks; the reviewing step grows.
- **Explaining as a skill**
  - Teaching something forces you to actually understand it. Explaining clearly is the same skill as prompting an AI well.
- **Basketball as data**
  - Every stat in our CSV is just a basketball event we count: points, rebounds, assists, steals, blocks, turnovers.
